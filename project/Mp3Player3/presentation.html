ㄲ Mp3Player3 발표자료

Mp3Player3 앱 발표자료

로컬 음악 재생, 플레이리스트 관리, 좋아요 기반 재생을 중심으로 구현한 안드로이드 MP3 플레이어

프로젝트: Mp3Player3 핵심 화면: Song / Playlist / Favorite / Player 발표 포인트: 재생흐름 · 셔플 · 반복 · 좋아요

1) 프로젝트 목표

  • 기기 내 음악 파일을 쉽게 탐색하고 재생
  • 플레이리스트 단위로 곡을 관리하고 연속 재생
  • 자주 듣는 곡은 좋아요로 빠르게 접근
  • 재생 중 셔플/반복으로 청취 경험 개선

2) 기술 구성

  • Language: Kotlin
  • UI: XML + ViewBinding + Fragment + RecyclerView
  • Playback: MediaPlayer 래퍼 (MusicPlayer)
  • State: LiveData / ViewModel / Repository 패턴 일부 적용

3) 핵심 기능

  • 곡 목록 재생: 곡 선택 시 PlayerActivity로 이동
  • 플레이리스트 관리: 생성, 선택, 곡 추가, 목록 재생
  • 좋아요 기능: 재생 화면에서 토글, Favorite 화면 자동 반영
  • 재생 제어: 재생/일시정지, 이전/다음, 시크바
  • 셔플: 다음 곡 선택을 현재 재생목록 내 랜덤으로 결정
  • 반복: 활성화 시 곡 종료 후 같은 곡을 처음부터 재생

4) 구현 파일 포인트

  • ui/PlayerActivity.kt
  • player/MusicPlayer.kt
  • ui/fragments/PlaylistFragment.kt
  • ui/fragments/FavoriteFragment.kt
  • repository/FavoritesRepository.kt

5) 재생 흐름 (발표 핵심)

기본 흐름: 한 곡 종료 → 다음 곡 재생 → 재생목록 마지막 곡 종료 시 재생 중지
셔플 버튼: 다음 곡을 현재 재생목록에서 랜덤 인덱스로 선택
반복 버튼: 활성화 시 곡 종료 때 다음 곡으로 이동하지 않고 현재 곡 재시작
좋아요 버튼: 현재 곡을 즐겨찾기 저장소에 반영, Favorite 화면이 즉시 갱신

6) 데모 시나리오

  1. Song 탭에서 곡 선택 후 Player 진입
  2. 기본 재생흐름: 곡 종료 시 다음 곡으로 넘어감 확인
  3. 셔플 ON 후 다음 곡 랜덤 전환 확인
  4. 반복 ON 후 현재 곡 반복 재생 확인
  5. 좋아요 누른 뒤 Favorite 탭에서 곡 반영 확인

7) 미구현/제약 사항

필수 공지:

  1. 앨범,아티스트부분은 androidstudio에서 앨범과 아티스트가 안읽히는 문제때문에 구현불가

보완 방향: 권한/MediaStore 쿼리/미디어 스캔 재검증 후 앨범·아티스트 뷰를 추가할 예정입니다.

8) 발표 멘트 예시 (1분 버전)

"Mp3Player3는 로컬 음악 재생에 집중한 앱입니다. 사용자는 곡 목록에서 바로 재생할 수 있고, 플레이리스트 단위로 곡을 구성해 원하는 목록만 들을 수 있습니다. 재생 화면에서는 셔플과 반복을 제공하는데, 셔플은 다음 곡을 현재 목록에서 랜덤으로 고르고, 반복은 활성화 시 현재 곡이 끝나면 같은 곡을 다시 재생합니다. 또한 좋아요 버튼으로 선호 곡을 저장하면 Favorite 화면에 즉시 반영됩니다. 다만 현재는 개발환경에서 앨범/아티스트 메타데이터를 안정적으로 읽지 못해 해당 화면은 구현하지 못했습니다."
