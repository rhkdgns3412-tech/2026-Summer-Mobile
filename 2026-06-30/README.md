# 📘 Kotlin 문법 공부 정리 노트 (이론편)

**학습 날짜:** 2026년 6월 30일  
**주요 주제:** 코틀린의 여러 가지 클래스 활용(상속, 다형성, 추상 클래스, 인터페이스, 클래스 위임) 및 람다 함수 학습

---

## 1. 다형성과 부모 타입 참조 (Polymorphism)
부모 클래스 타입의 저장 공간(리스트, 변수 등)에 자식 클래스로 만든 객체들을 섞어서 보관하고 다룰 수 있는 성질입니다.

* **접근 자격(무엇을 쓸 수 있는가):** 겉을 감싸고 있는 **부모 타입**이 결정합니다. 부모 클래스 참조 변수에 자식 객체가 할당되면 컴파일러 눈에는 부모의 기능만 보이므로, 자식만의 고유 메서드는 바로 호출할 수 없습니다.
* **실제 행동(어떻게 작동하는가):** 내부에 담긴 **실제 자식 객체**가 결정합니다. 겉보기엔 부모의 공통 메서드를 호출하는 것처럼 보여도, 실행 시에는 자식이 재정의한 ‘오버라이드(`override`) 메서드’가 동작합니다.
* **유연한 흐름 제어 (`when`):** 객체의 실제 타입에 따라 분기 처리를 할 때 `when`문과 `is` 키워드를 조합하여 조건문보다 훨씬 직관적인 다형성 처리가 가능합니다.

---

## 2. 타입 검사와 스마트 캐스트 (Type Check & Smart Cast)
부모 타입의 포장지 안에 들어있는 객체의 진짜 정체를 파악하고, 자식 클래스만의 고유 기능을 안전하게 사용하기 위한 단계입니다.

* **타입 검사 (`is`):** 객체의 실제 정체가 특정 자식 클래스에서 생성된 것이 맞는지 논리적으로 검사합니다.
* **명시적 형변환 (`as`):** 넓은 범위(부모)에서 좁은 범위(자식)로 타입을 강제 다운캐스팅(Downcasting)하는 기능입니다.
* **스마트 캐스트 (Smart Cast):** 코틀린의 영리한 기능으로, `is`를 통해 타입 검사가 한 번 성공하면 그 영역 안에서는 개발자가 굳이 `as`로 형변환을 하지 않아도 컴파일러가 해당 자식 타입으로 자동 인정해 줍니다.

## 3. 추상 클래스 vs 인터페이스 (괄호 `()`의 유무)
자식 클래스에게 특정 규격과 설계를 강제한다는 점은 같으나, 본질과 상속 방식에서 뚜렷한 차이가 있습니다.

### 1) 추상 클래스 (`abstract class`)
* **관계:** **IS-A** (~는 ~이다)의 관계로, 자식들의 공통된 본질과 혈통을 묶어줍니다. `open` 키워드가 없어도 기본적으로 상속이 가능합니다.
* **특징:** 추상 메서드가 포함된 클래스는 반드시 추상 클래스로 선언해야 합니다. 추상 클래스 안의 일반 메서드를 자식이 오버라이드하려면 부모 메서드 앞에 `open` 키워드를 명시해야 합니다.
* **문법적 특징:** 클래스이므로 자식이 메모리에 생성될 때 부모의 **생성자(Constructor)를 반드시 호출**해야 합니다. 부모에게 생성자가 생략되어 있더라도 상속 선언 시 부모 이름 뒤에 **괄호`()`**를 필수로 붙여야 합니다. (단일 상속만 가능)

### 2) 인터페이스 (`interface`)
* **관계:** **CAN-DO** (~를 할 수 있는)의 관계로, 본질과 상관없이 특정 기능이나 자격을 부여합니다.
* **특징:** 구현부가 없는 '추상 메서드'뿐만 아니라, 본문(바디)이 구현되어 있는 '기본 메서드(Default)'를 모두 가질 수 있습니다. 자식 클래스에서 기본 메서드를 원하는 대로 오버라이드하여 덮어쓰는 것도 가능합니다.
* **문법적 특징:** 생성자라는 개념 자체가 아예 존재하지 않습니다. 따라서 구현(상속) 선언 시 인터페이스 이름 뒤에 **괄호`()`를 붙이면 컴파일 에러**가 발생합니다. (다중 구현 가능)

## 4. 클래스 위임 (Class Delegation)
다른 클래스의 기능을 재사용하고 싶을 때, 상속의 단점(부모와 자식 간의 과도한 결합)을 피하기 위해 사용하는 고급 설계 기법입니다.

* **원리:** 내가 직접 모든 기능을 구현하거나 상속받는 대신, 그 일을 이미 완벽하게 처리해 둔 '대리인(다른 객체)'을 내부에 두고 일을 전부 떠넘기는 방식입니다.
* **코틀린의 장점 (`by`):** 원래 타 언어에서는 대리인에게 일을 토스하는 전달용 코드를 개발자가 일일이 작성해야 했습니다. 코틀린은 `by` 키워드 하나로 이 위임 구조를 컴파일러가 무대 뒤에서 자동으로 연결해 줍니다.
* **결과:** 개발자는 대리인의 기능을 마치 자기 것처럼 유연하게 호출하면서, 동시에 자신만의 고유한 기능(함수)만 깔끔하게 추가하여 확장할 수 있습니다.

## 5. 람다 함수와 고차 함수 (Lambda Expression)
함수를 하나의 독립된 값이나 변수처럼 다룰 수 있게 해주는 현대적이고 간결한 문법 스타일입니다.

* **익명 함수와 변수화:** 이름이 없는 함수로, 중괄호 `{}` 안에 코드 블록을 작성하여 변수에 담거나 다른 함수의 인자(매개변수)로 직접 넘겨줄 수 있으며 함수 자체를 반환할 수도 있습니다.
* **단일 매개변수의 생략 (`it`):** 람다 함수로 전달되는 매개변수(Argument)가 딱 한 개뿐인 경우, 변수 이름을 일일이 선언할 필요 없이 코틀린이 제공하는 시스템 예약어인 **`it`** 하나로 축약해서 가리킬 수 있습니다.
* **함수 타입 표현:** `(입력 타입) -> 반환 타입` 형태로 함수의 규격을 명확히 선언할 수 있으며, 반환값이 없는 경우 `Unit` 타입을 사용합니다. 반환 타입에 또 다른 함수 형태(`() -> String`)를 지정하여 함수를 리턴하는 강력한 구조를 만들 수도 있습니다.
